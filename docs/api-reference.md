# API Reference

Base URL (local): `http://localhost:8000`

All routes are prefixed with `/api`. Every route except `/` and `/api/auth/login`
requires a bearer token:

```
Authorization: Bearer <token-from-login>
```

Tokens are obtained from `POST /api/auth/login` and expire after `JWT_TTL_HOURS`
(default 12). Interactive docs are also auto-generated by FastAPI at
`http://localhost:8000/docs`.

---

## Health

### `GET /`
Liveness check. No auth.
```json
{ "status": "ok", "service": "SupplyLens API" }
```

---

## Auth

### `POST /api/auth/login`
Exchange credentials for a token. **No auth required.**

Request:
```json
{ "email": "admin@supplylens.io", "password": "demo1234" }
```
Response:
```json
{ "token": "<jwt>", "user": { "email": "...", "name": "...", "role": "admin", "tenant_id": "demo" } }
```

### `GET /api/auth/me`
Returns the current user derived from the token.

## Today & decisions

### `GET /api/today`
Daily risk summary: critical/high counts, total dollars at risk, and the top
at-risk cards.

### `GET /api/explain-decision?sku={sku}&site={site}`
Ranked fix actions (rebalance / expedite / substitute / defer) for one SKU at one
site, each with cost, benefit, and confidence. `404` if the SKU/site is unknown.

---

## AI copilot

### `GET /api/ai/briefing`
A 2-sentence morning briefing plus the top cards. `ai: true` when the live model
was used, `false` when the deterministic fallback was used.

### `GET /api/ai/explain?sku={sku}&site={site}`
Natural-language explanation of why a SKU is at risk and which action to take.

### `POST /api/ai/ask`
Free-form question grounded on the live risk + supplier data.
```json
{ "question": "Which suppliers are causing the most risk right now?" }
```

## Suppliers

### `GET /api/suppliers`
All active suppliers with on-time rates, quality scores, and incident counts.

### `GET /api/incidents?supplier_id={id}`
Supplier incidents, optionally filtered to one supplier.

## Inventory engine

### `GET /api/inventory/stats`
Headline inventory KPIs.

### `GET /api/inventory/parts`
All parts with computed metrics. Optional filters: `abc`, `vendor`,
`status` (`over` | `under`).

### `GET /api/inventory/part/{part_number}`
Full 52-week simulation detail for one part. `404` if not found.

### `GET /api/inventory/part/{part_number}/demand`
Seasonal demand forecast and forecast-adjusted runway for one part.

### `GET /api/inventory/worst?kind={over|under}&limit={n}`
Worst-offender parts ã most overstocked or most understocked.

### `GET /api/inventory/warehouse-load`
Projected total pallet load per week across all parts.

### `POST /api/inventory/import`
Upload a parts file (`multipart/form-data`, field `file`). Accepts `.csv`,
`.xlsx`, `.xlsm`. Returns `{ "imported": <count> }`. See
[Using Your Own Data](data-guide.md#method-a--import-a-parts-file-no-code) for the
column format.

## Hedging planner

### `GET /api/hedging/scenario`
Hedged-vs-spot cost and "Amount Saved" for a strategy blend. Query params control
the split (e.g. Strategy 1 vs Strategy 2 percentages).

### `GET /api/hedging/forecast`
The commodity price forecast used to feed the hedging pricing.

---

## Actions (execution)

All action routes are **tenant-scoped** by the token's `tenant_id`.

### `POST /api/actions/create-po`
Create an expedite purchase order.
```json
{ "sku": "SKU001", "site": "BOSTON", "label": "Expedite PO", "cost": 1200, "benefit": 8000 }
```

### `POST /api/actions/transfer`
Create a stock transfer between sites (same body shape as above).

### `GET /api/actions/status`
The log of actions for the current tenant.
